  <action>
Update src/pages/ClientView.jsx to add proper ARIA roles for tab navigation pattern.

**Step 1: Update tab navigation container (line 772)**
Find the `<nav>` element wrapping tab buttons and ensure it has:
- `role="tablist"` attribute
- Keep existing `aria-label="Tabs"` attribute

**Step 2: Update each tab button (lines 773-777)**
Add to each of the 5 tab buttons (summary, property, activities, flights, transportation):
- `role="tab"`
- `aria-selected={activeTab === 'summary'}` (boolean, not string - use expression matching the tab name)
- `aria-controls="summary-panel"` (unique ID matching the panel, e.g., "summary-panel", "property-panel", etc.)
- `id="summary-tab"` (unique ID for the tab button itself)

Example for summary tab button:
```jsx
<button
  onClick={() => setActiveTab('summary')}
  role="tab"
  aria-selected={activeTab === 'summary'}
  aria-controls="summary-panel"
  id="summary-tab"
  className={`whitespace-nowrap py-4 px-3 border-b-2 font-medium text-sm sm:text-base flex items-center ${activeTab === 'summary' ? 'border-yellow-500 text-yellow-600' : 'border-transparent text-gray-500 hover:text-gray-700 hover:border-gray-300'}`}
>
  <ClipboardList size={18} className="mr-2" /> Summary
</button>
```

Repeat for all 5 tabs with appropriate IDs:
- summary: aria-controls="summary-panel", id="summary-tab"
- property: aria-controls="property-panel", id="property-tab"
- activities: aria-controls="activities-panel", id="activities-tab"
- flights: aria-controls="flights-panel", id="flights-tab"
- transportation: aria-controls="transportation-panel", id="transportation-tab"

**Step 3: Update tab panel containers (lines 781+)**
Add to each tab panel's outer `<div>`:
- `role="tabpanel"`
- `id` matching the aria-controls value (e.g., "summary-panel")
- `aria-labelledby` pointing back to the tab button ID (e.g., "summary-tab")

Example for summary panel:
```jsx
{activeTab === 'summary' && (
  <div
    role="tabpanel"
    id="summary-panel"
    aria-labelledby="summary-tab"
    className="bg-white rounded-2xl shadow-xl p-4 sm:p-6 font-sans border border-gray-100 text-left"
  >
    {/* existing content */}
  </div>
)}
```

Repeat for all 5 tab panels with matching IDs and aria-labelledby references.

**Important:** The conditional rendering (`activeTab === 'summary'`) is fine - hidden panels don't need `aria-hidden` because they're not in the DOM. Only add ARIA attributes to the actual rendered panel.

**Step 2: Add aria-live to PriceSummaryPanel.jsx**

Open src/components/PriceSummaryPanel.jsx (created in Phase 1).

Find the final price display section (should be around line 99 in the plan template, showing "FINAL PRICE" label and finalQuote value).

Wrap the final price row in a div with aria-live="polite" and aria-atomic="true":

```jsx
<div aria-live="polite" aria-atomic="true" className="sr-only">
  Your total is now {displayPrice(finalQuote, selectedCurrency)}
</div>

<div className="flex justify-between items-baseline">
  <span className="text-xl font-bold text-gray-700">FINAL PRICE</span>
  <span className="text-2xl font-bold text-blue-700">
    {displayPrice(finalQuote, selectedCurrency)}
  </span>
</div>
```

The sr-only div provides screen reader announcements when the price changes, while the visible div shows the price visually. Use Tailwind's sr-only class: `sr-only` is `position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px; overflow: hidden; clip: rect(0,0,0,0); white-space: nowrap; border-width: 0;`

If sr-only doesn't exist in Tailwind config, add the aria-live directly to the visible element:
```jsx
<div aria-live="polite" aria-atomic="true" className="flex justify-between items-baseline">
  <span className="text-xl font-bold text-gray-700">FINAL PRICE</span>
  <span className="text-2xl font-bold text-blue-700">
    {displayPrice(finalQuote, selectedCurrency)}
  </span>
</div>
```
